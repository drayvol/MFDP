# Домашнее задание №2

## Прототип продукта

Проект: **Telegram-бот для автоматической проверки решений тригонометрических уравнений ЕГЭ по фотографии**

## 1. Ключевая гипотеза

Если мы предложим Telegram-бота для автоматической проверки пункта а задания №13 ЕГЭ ученикам 10-11 классов и преподавателям математики, то время получения обратной связи по решению сократится с нескольких часов до пяти минут при сохранении качества проверки не ниже **F1-score  0.80**, поскольку преподаватель не всегда может проверить работу сразу, а система выполняет первичную проверку автоматически.

**Сегмент:** ученики 10-11 классов и преподаватели математики.

**Сценарий:** автоматическая проверка решения вместо ожидания ответа преподавателя.

## 2. Ключевые метрики

| Метрика | Что измеряет | Целевое значение | Способ измерения | Частота измерения |
| --- | --- | --- | --- | --- |
| Время получения обратной связи | Время от отправки фотографии до получения результата проверки | <= 5 минут | Логи Telegram-бота | Ежедневно |
| Доля положительных оценок ответа | Пользователь считает объяснение ошибки полезным | >= 60% | Кнопка "Полезно / Не полезно" | Еженедельно |
| CER | Качество OCR при распознавании математических выражений | <= 10% | Сравнение с эталонным LaTeX | После OCR-тестов |
| F1-score | Качество определения "верно / есть ошибка" | >= 0.80 | Сравнение с экспертной разметкой на golden set | После экспериментов |

## 3. Скоуп прототипа

### Входит в прототип

- Фото решения -> Telegram.
- OCR -> LaTeX.
- SymPy проверяет пункт а задания №13: верно или неверно.
- LLM объясняет ошибку.
- Система выставляет оценку:
  - **1 балл** - решение корректно;
  - **0 баллов** - решение содержит ошибку.
- Оценка полезности ответа пользователем: "полезно / не полезно".

### Не входит в прототип

- Проверка пункта б задания №13.
- Проверка других заданий ЕГЭ.
- Обучение собственной OCR-модели.
- Обучение собственной LLM.
- Персональная статистика ученика.
- Платежная система.
- Личный кабинет пользователя.
- Рекомендации похожих задач.
- Адаптивный план обучения.

Эти функции не влияют на проверку основной гипотезы и могут быть добавлены после подтверждения работоспособности базового сценария.

## 4. Готовые блоки и инструменты

| Компонент             | Инструмент                                                                                                                                                                                     |
|-----------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Интерфейс             | Telegram Bot API                                                                                                                                                                               |
| OCR                   | pix2tex / TrOCR / TexTeller                                                                                                                                                                    |
| Датасет для OCR рукописных почерков             | CROHME (Competition on Recognition of Online Handwritten Mathematical Expressions,рукописные математические выражения) - открытый датасет для сравнения OCR моделей, Im2LaTeX-100k, Golden set |
| Символьная математика | SymPy                                                                                                                                                                                          |
| Генерация объяснений  | LLM                                                                                                                                                                                            |
| Хранение данных       | PostgreSQL                                                                                                                                                                                     |
Для выбора OCR-модели используется датасет CROHME, так как он содержит рукописные математические выражения и подходит для проверки способности модели распознавать почерк и переводить формулы в структурированный вид.

Однако CROHME используется только для OCR-бенчмаркинга. Для проверки всего прототипа нужен отдельный golden set из решений задания №13 ЕГЭ, потому что продукт должен анализировать не отдельную формулу, а полный ход решения ученика.

Golden set будет состоять минимум из 200 решений:
- каждое решение размечается преподавателем: 1 балл / 0 баллов, место ошибки, корректное объяснение.
### Что реализуется самостоятельно

Самостоятельно разрабатываются:

- логика проверки решения;
- интеграция OCR и SymPy;
- промпты для LLM;
- механизм выставления оценки;
- Telegram-бот;
- система оценки качества.

Основная ценность проекта заключается в построении полного пайплайна автоматической проверки математических решений.

## 5. Главный риск, который проверяется прототипом

Главный риск проекта:

**Сможет ли система корректно определять наличие ошибки в решении тригонометрического уравнения по фотографии и выдавать результат, сопоставимый с проверкой преподавателя?**

Этот риск состоит из двух частей:

1. OCR может некорректно распознать математическое выражение.
2. Алгоритм проверки может ошибочно определить корректность решения.

Прототип должен показать, возможно ли достичь качества проверки, достаточного для использования системы в качестве инструмента первичной проверки.

## 6. Конечный формат прототипа

Конечным результатом является **Telegram-бот**.

Пользовательский сценарий:

```text
Фото решения -> OCR -> SymPy -> LLM -> Оценка решения
```

### Пример ответа при ошибке

```text
Оценка: 0 баллов

В решении обнаружена ошибка.
Неверно выполнено преобразование на шаге ...
Правильное преобразование:
...
```

### Пример ответа при корректном решении

```text
Оценка: 1 балл

Решение пункта а выполнено корректно.
Ошибок не обнаружено.
```

## 7. Критерии остановки теста прототипа
**Длительность теста:** 2 недели. Если за 2 недели не набрано минимальное количество решений — продлеваем до 4 недель максимум.

**Минимальный объём данных для OCR:**  1000 рукописных примеров из датасета CROHME для бенчмаркинга модели. Собственный golden set (200 фото рукописных решений) используется для финальной оценки на целевом почерке.

Тест считается успешным, если одновременно выполнены следующие условия:

- протестировано не менее 100 пользовательских решений;
- среднее время получения ответа не превышает 1 минуты;
- F1-score на golden set не ниже 0.80;
- доля завершенных проверок не ниже 80%;
- OCR демонстрирует CER не выше 10%.

Гипотеза считается опровергнутой, если:

- **F1 < 0.60** после перебора разных LLM и промптов - пайплайн технически нежизнеспособен;
- OCR не дает пригодный LaTeX даже после смены модели - входные данные не поддаются обработке.
