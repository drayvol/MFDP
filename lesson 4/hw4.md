## 1. Источники и состав данных

В проекте используются три независимых набора данных, каждый из которых решает отдельную задачу в общей архитектуре системы.

### 1.1 Golden Set

**Источник:** собственный датасет автора проекта.

**Назначение:**  оценка качества системы проверки решений.

Состав датасета:

- 200 экспертно проверенных решений задания №13 профильного ЕГЭ;
- фотографии рукописных решений;
- бинарная оценка (`score`):
  - 1 — решение верное;
  - 0 — решение неверное;
- комментарий эксперта при наличии ошибки;
- тип тригонометрического уравнения.

Данный набор используется только для финального тестирования системы и не участвует в выборе OCR-модели.

---

### 1.2 CROHME 2013/2014/2016

**Источник:** Competition on Recognition of Online Handwritten Mathematical Expressions.

**Назначение:** сравнение OCR-моделей.

Датасет содержит:

- изображения рукописных математических выражений;
- эталонную LaTeX-разметку.

На данном наборе были протестированы три OCR-модели:

- pix2tex;
- Texify;
- Uni-MuMER.

Оценка проводилась по метрике Character Error Rate (CER).

---

### 1.3 MATH Dataset

**Источник:** https://huggingface.co/datasets/sarahpann/MATH.

Из полного датасета были отобраны только задачи, содержащие математические уравнения.

Всего было сформировано 70 задач.

Данный набор используется для:

- разработки алгоритма проверки;
- тестирования SymPy;
- получения эталонных решений.

# 2. Базовый EDA

## 2.1 Golden Set

Golden Set содержит экспертно размеченные решения реальных учеников.

Основные особенности:

- представлены как правильные, так и ошибочные решения;
- присутствуют комментарии экспертов;
- решения относятся к различным типам тригонометрических уравнений.

Такой набор позволяет объективно оценивать качество автоматической проверки.
| Метрика | Значение |
|---------|----------|
| Всего примеров | 200 |
| Score = 1 (верно) | 169 (84.5%) |
| Score = 0 (неверно) | 31 (15.5%) |
| Примеров с комментарием | 31 из 31 (100%) |

**Вывод для моделирования:** баланс классов 84.5% / 15.5% отражает реальное распределение решений ЕГЭ — верных решений ожидаемо больше. При валидации необходима стратификация по score, чтобы сохранить это соотношение в val и test.

#### Распределение по типу метода решения

| Тип уравнения | Всего | Верных | Неверных | Доля ошибок |
|---------------|------:|-------:|---------:|------------:|
| Разложение на множители | 137 | 117 | 20 | 14.6% |
| Замена | 35 | 30 | 5 | 14.3% |
| Смешанное | 28 | 22 | 6 | 21.4% |

**Вывод:** датасет преимущественно состоит из задач на разложение на множители (68.5%). Смешанные уравнения имеют наибольшую долю ошибок (~21%), что соответствует их сложности.

#### Типы ошибок (score = 0)

| Тип ошибки | Количество |
|------------|----------:|
| Неверное решение тригонометрического уравнения (sin/cos/tan) | 14 |
| Прочее (потеря коэффициента, неверная замена, неверные ограничения и др.) | 8 |
| Ошибка в формуле приведения | 3 |
| Вычислительная ошибка | 3 |
| Нет ответа | 2 |
| Ошибка в ОДЗ | 1 |

**Вывод:** большинство ошибок связаны с тригонометрическими уравнениями. Чекер должен уметь обнаруживать ошибки именно в этих шагах решения.

## 2.2 Сравнение OCR-моделей

В качестве основной метрики использовался Character Error Rate (CER).

![](/lesson%204/ocr-benchmark/reports/ocr_models_comparison.png)

По результатам эксперимента получены следующие значения.

| Модель | Средний CER | Медианный CER |
|---------|------------:|--------------:|
| pix2tex | 97.7% | 100% |
| Texify | 32.2% | 16.7% |
| Uni-MuMER | **10.4%** | **0%** |

Также была рассчитана доля изображений, распознанных с приемлемым качеством.

- CER < 10%
- CER < 20%

Из результатов видно, что Uni-MuMER существенно превосходит остальные модели.

Более 70% изображений распознаются с CER ниже 10%, а более 80% — с CER ниже 20%.

На основании данного сравнения в проект была выбрана модель **Uni-MuMER**.

# 3. Оценка качества разметки

Разметка Golden Set выполнялась вручную преподавателем математики.

Каждая работа содержит:

- итоговую бинарную оценку;
- экспертный комментарий при наличии ошибки.

Таким образом, качество разметки можно считать высоким, поскольку:

- отсутствуют автоматически сгенерированные метки;
- все решения проверены вручную;
- имеется объяснение ошибок.



### Возможные направления улучшения

В дальнейшем качество разметки может быть повышено за счет:

- увеличения числа ошибочных решений;

# 4. Формирование выборки

Каждый используемый датасет решает собственную задачу.

| Датасет                          | Назначение |
|----------------------------------|------------|
| CROHME                           | сравнение OCR-моделей |

Такое разделение позволяет избежать утечки информации между этапами разработки и оценки модели.

# 5. Стратегия валидации

Для различных компонентов системы используются независимые наборы данных.

1. На датасете **CROHME** производится сравнение OCR-моделей и выбор наиболее качественной модели распознавания.

2. На датасете **MATH** и валидационной части **Golden set** разрабатывается алгоритм проверки математических решений и тестируется работа SymPy.

3. Тестовая часть **Golden Set** используется исключительно для итоговой оценки качества всей системы и не участвует в выборе OCR-модели или разработке алгоритма проверки.

Подобная стратегия обеспечивает объективную оценку качества конечной системы и исключает влияние тестовых данных на процесс разработки.
Golden Set разбивается на val и test стратифицированно по score:

| Split | Доля | Примеров | Назначение |
|-------|-----:|---------:|------------|
| val | ~70% | ~140 | Разработка и отладка системы, подбор эвристик |
| test | ~30% | ~60 | Финальная оценка, используется один раз |

Стратификация по score сохраняет соотношение 84.5% / 15.5% в обоих сплитах.

**Метрики оценки чекера:** F1-score (основная)
