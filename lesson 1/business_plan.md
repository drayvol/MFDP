# Бизнес-план проекта

## Telegram-бот для автоматической проверки решений тригонометрических уравнений ЕГЭ по фотографии

## 1. Бизнес-анализ

### Описание проекта

Проект направлен на разработку Telegram-бота, который автоматически проверяет решения тригонометрических уравнений ЕГЭ по фотографии.

Бот должен помогать ученикам и преподавателям ускорить первичную проверку решений задания №13, давать мгновенную обратную связь и предлагать похожие задачи для дополнительной практики.

### Заказчик и пользователи

**Заказчик:** учитель или репетитор по математике или образовательная организация, осуществляющая подготовку учащихся к ЕГЭ.

**Пользователи:**

- ученики 10-11 классов;
- учителя математики;
- репетиторы и онлайн-школы.

### Бизнес-проблема

Во время подготовки к ЕГЭ ученики регулярно отправляют фотографии решений задач для проверки. Проверка решений требует значительных временных затрат со стороны преподавателя и не всегда может быть выполнена оперативно.

Из-за этого ученик может долго ждать обратной связи, а преподаватель тратит много времени на однотипную первичную проверку решений.

### Бизнес-цель

Цель проекта - сократить время первичной проверки решений и обеспечить ученику мгновенную обратную связь по допущенным ошибкам.

Планируется создать Telegram-бота, который:

- принимает фото решения задания №13 по тригонометрии;
- распознает решение через OCR и преобразует его в LaTeX;
- проверяет пункт а и определяет, верно ли решение: 1 балл или 0 баллов;
- объясняет ошибку, если она есть;
- предлагает похожую задачу для практики.

### Существующие решения и их недостатки

| Решение | Недостаток |
| --- | --- |
| Репетитор | Дорого, недоступен 24/7 |
| Photomath / Wolfram | Дают ответ, но не проверяют ход решения |
| ChatGPT напрямую | Нет структуры и привязки к критериям ЕГЭ |
| Решебники | Нет обратной связи по конкретной ошибке |

Разрабатываемое решение ориентировано именно на формат ЕГЭ. Оно позволяет автоматически анализировать решение ученика, проверять корректность решения пункта а задания №13, выставлять оценку 0 или 1 балл и объяснять допущенные ошибки.

## 1.1. Текущая ситуация

### Доступные данные

Для реализации проекта доступны:

- около 200 экспертно размеченных решений тригонометрических уравнений;
- открытые банки заданий ЕГЭ;
- открытые варианты ФИПИ;
- открытые математические датасеты:
  - CROHME;
  - Im2LaTeX;
  - latex-formulas;
  - MATH-500;
  - PyCoT-Trigonometry.

Планируется расширить собственный набор данных за счет открытых источников.

Данные будут разделены на два набора:

- **validation set** - для выбора моделей и настройки системы;
- **golden set** - для финальной оценки качества всего пайплайна.

### Технические ресурсы

Для реализации проекта планируется использовать:

- Telegram Bot API;
- OCR-модель для преобразования фотографии в LaTeX;
- LLM-модель для анализа решения и генерации объяснений;
- SymPy для символьной проверки математической корректности;
- PostgreSQL для хранения данных;
- векторную базу для поиска похожих задач.

Дополнительная закупка оборудования не требуется.

### Основные риски

| Риск | Проблема | Снижение риска |
| --- | --- | --- |
| Ошибки OCR | Некорректное распознавание математических выражений по фотографии | Использование специализированных OCR-моделей для математических выражений; предварительное тестирование OCR на открытых бенчмарках |
| Ошибки интерпретации решения | LLM может некорректно интерпретировать решение или допустить ошибку при проверке | Использование SymPy для проверки математической корректности; сравнение нескольких архитектур проверки; тестирование на независимом наборе данных |
| Недостаточный объем данных | Ограниченное количество экспертно размеченных решений | Использование открытых математических датасетов, банков задач ЕГЭ и постепенное накопление пользовательских решений |

## 1.2. Решаемые задачи с точки зрения аналитики

Проект состоит из трех основных аналитических задач.

### Задача 1. OCR-бенчмаркинг

**Цель:** выбрать лучшую модель для распознавания рукописных формул и преобразования изображения в LaTeX.

**Модели для сравнения:**

- pix2tex;
- TrOCR;
- TexTeller.

**Датасеты:**

- CROHME;
- Im2LaTeX;
- latex-formulas, включая около 550 000 пар "формула - изображение".

**Метрики:**

- Основная: CER, Character Error Rate;
- Дополнительно: Exact Match.

**Порог успеха:** CER <= 10%.

### Задача 2. Проверка решения

**Цель:** проверить, может ли пайплайн SymPy + LLM корректно находить ошибки в пункте а задания №13.

**Этап 1:** проверка работоспособности пайплайна на открытых датасетах PyCoT-Trigonometry и MATH-500.

**Этап 2:** финальная оценка всего пайплайна на golden set из 200 примеров.

**Метрики:**

- Основная: F1;
- Дополнительные: Accuracy, FNR;

**Порог успеха:**

- F1-score ≥ 0.80

### Задача 3. Поиск похожих задач

**Цель:** подобрать пользователю похожую задачу для закрепления материала после проверки решения.

**Метод:**

- получение эмбеддингов задач;
- поиск по косинусной близости.

**Датасеты:**

- база задач ЕГЭ;
- PyCoT-Trigonometry;
- собственный набор задач.

**Метрики:**

- Recall@5.

### Финальная оценка системы

После выбора OCR-модели и архитектуры проверки будет проведена оценка полного пайплайна:

```text
Фото решения -> OCR -> SymPy + LLM -> Оценка -> Рекомендация задач
```
Финальная оценка проводится на независимом golden set, который не использовался при выборе моделей и настройке системы.
